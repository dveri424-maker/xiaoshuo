`爽文四线压场与多女主感情模型.md` 是权力、武力、财力、多女主感情/异性吸引力四线的读者爽感执行模型，用于把爽点落成动作、物件、站位、关系余温和后果。多女主感情线本身就是爽点，不是资源工具；它不单独改写剧情事实，不覆盖卷纲和章节细纲。若四线模型牵动具体章节收益、关键物件、人物关系或时间线，必须同步修改对应卷纲或细纲。

`第二卷章节细纲.md` 是第 041-120 章正式细纲，负责把第二卷防偏口径落成五个大案团、逐章冲突、关键物件、可见收益和多女主关系推进。它只管第二卷章法，不重写第一卷已落地事实；若它与 `新版正文第二十一至四十章合规复盘与第二卷防偏校准.md` 在第二卷具体章节安排上有细节差异，以 `第二卷章节细纲.md` 为准。根目录旧版 `第二卷章节细纲.md` 只可作为旧资产池，不得作为新版正史。

`新版正文前二十章合规复盘与防偏校准.md` 只约束执行方向，不新增或覆盖 `第一卷章节细纲.md` 的章节收益、关键物件、出场人物和章末钩子；如确需改变章节收益，必须同步修改 `第一卷章节细纲.md`。

`新版正文第二十一至四十章合规复盘与第二卷防偏校准.md` 是第 21-40 章正文定稿后的成果校准和第二卷防偏文档，负责细化第一卷卷末实际落地收益、未落地保留项和第二卷开局限制；它不单独覆盖上位文档。若正文复盘牵动章节收益、关键物件、人物关系或时间线，必须同步修改总则、新设定、卷纲或章节细纲。

`新版正文主角核心视角与读感校准.md` 只负责正文入口、主角视角锚点、环境描写和读感改稿手法，不新增设定边界，不改变章节收益、人物关系和时间线。合规复盘负责边界，主角核心视角校准负责手法；如发生冲突，以总则、新设定、卷纲、章节细纲和合规复盘为准。

`新版正文轻松爽感与老嗷读感校准.md` 只负责轻松爽感、发现型强、硬场面里的坏笑、银星异常使用口径和物件行为后果，不新增设定边界，不改变章节收益、关键物件和时间线。它是读感验收口径，不是正文打卡表；如重排正文牵动出场人物、关键物件、章节收益或章末钩子，必须同步修改对应细纲。

旧版正文和旧版设定不再作为新版正史。它们只能提供案件节奏、物件、多女主感情关系、场面站位、资源模块和升级爽点，不能反向修改新版正史边界。
