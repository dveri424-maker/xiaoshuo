# 活人感创作 Skill

本 skill 用于中文小说正文创作、续写和改稿，目标是让文本有活人感、代入感和主角贴身感。

它不负责新增世界观设定，也不替代章节细纲。使用时必须服从项目设定、卷纲、章节细纲、既有正文连续性，以及用户本轮明确要求。

## 核心目标

正文要让读者像贴着主角本人醒来、看见、误判、反应、做决定，而不是听作者站在旁边讲设定、讲局势、讲爽点。

优先做到：

1. 主角先进入读者手里；
2. 麻烦从主角感官和当场利害里砸进来；
3. 背景、身份、年代、规则随动作和记忆触发进入；
4. 主角的强由误判、动作和后果被读者发现；
5. 旁白带主角本人的脑子、脾气、经验和尴尬，不带作者解释腔。

## 强制规则

### 1. 用贴主角意识流动的第三人称

必须使用第三人称，但叙述距离贴近主角本人。

允许：

- 写主角先注意到的声音、气味、疼痛、物件、站位；
- 写主角当下的错愕、误判、职业本能、心里吐槽；
- 写主角因为某个物件、称呼、伤口、门声、枪、账本、尸体而触发记忆；
- 写主角在一边理解情况、一边处理麻烦。

避免：

- 作者站在高处介绍世界；
- 先讲完整设定，再让主角行动；
- 用旁白替读者总结“这说明”“这意味着”“所有人都意识到”；
- 长时间写主角看不见、听不见、不能立刻利用的信息。

### 2. 信息必须自然进入

设定信息只能从以下入口进入：

- 感官：痛、血味、尘土、枪油、冷意、声响、灯光；
- 动作：摸枪、扣腰带、下床、开门、上马、按住证物；
- 误判：主角以为还在原世界，随后被物件或称呼纠正；
- 记忆触发：碰到枪、马、徽章、账本、钥匙、尸体、工资箱等硬物件后，原身记忆碎片浮出；
- 当场利害：死人、钱、枪口、门槛、称呼、权力、证人、箱子归属。

禁止大段旁白硬讲：

- “他原本是什么人，现在穿越到了哪里”；
- “这个世界是什么规则”；
- “某势力的完整背景”；
- “本章要完成什么收益”。

### 3. 禁止出戏比喻和作者式总结

禁止任何可能让读者出戏的比喻句，尤其是为了显得有文采、酷、电影感而加的比喻。

重点警惕：

- 像、仿佛、如同、好像、似的、宛如、一半一半式比喻；
- 抽象化比喻：命运、棋局、风暴、巨兽、深渊、潮水等；
- 作者式总结：所有人都知道、这意味着、这一刻、从此、真正的、命运开始转动、读者能看出；
- 爽点总结：众人服了、全场安静了、主角完成了压场、收益落地。

可以写具体行为代替总结：

- 谁退了半步；
- 谁闭嘴；
- 谁改口；
- 谁把手从枪套上拿开；
- 谁把账本递过来；
- 谁把称呼换了；
- 谁敢笑、谁不敢笑。

### 4. 禁止高频短句短词堆酷感

不能靠连续短句、单词句、硬切镜头制造“冷硬”“装”“酷哥摆拍”。

禁止连续堆：

- “血。枪。死人。”
- “他醒了。门响。街上乱了。”
- “不对。很不对。”
- “箱子不走。人也不走。”

短句只能在真正的惊醒、判断落点、动作落点处少量使用。正文主体要用贴主角心理流动的自然长短句，让读者跟着主角理解和反应。

### 5. 旁白贴主角，不贴作者

旁白可以有吐槽、判断、尴尬、职业习惯和旧经验，但必须像主角自己脑子里冒出来的。

允许：

- “刑警干久了，换个地方也改不了这毛病。”
- “眼下显然不是感慨身体变强的时候。”
- “没有徽章，还派人来叫他压死人和钱。”

避免：

- “这一幕体现了主角的专业能力。”
- “这场冲突完成了主角的街面立足。”
- “读者此时会明白……”
- “这是主角获得收益的关键节点。”

## 开章推荐结构

适合穿越、案件、权力、爽文开章：

1. 主角因疲惫、伤痛、睡眠、酒醒、落水、昏迷等状态醒来；
2. 外部麻烦砸门、喊人、抢物、死人、枪响或账本出事；
3. 主角发现身体、环境、称呼、物件不对；
4. 主角触碰关键物件，记忆碎片进入；
5. 主角只理解当前麻烦所需的信息，不一次性读完设定；
6. 主角带着不适、误判和职业本能进入现场；
7. 现场破局靠动作、证据、物件和旁人站位变化；
8. 章末留下一个具体物件或下一步麻烦。

## 改稿检查清单

改稿后逐项检查：

1. 前三段是否已经贴住主角身体、感官或当场麻烦？
2. 设定信息是否从动作、物件、记忆触发或当场利害进入？
3. 是否有作者式总结、爽点总结或读者提示句？
4. 是否有出戏比喻句？
5. 是否连续使用短句短词堆酷感？
6. 旁白是否像主角本人在想，而不是作者在讲？
7. 主角的强是否由别人误判、主角动作和现场后果体现？
8. 场面结束后，是否能看见具体行为变化，而不是抽象说“众人服了”？

## 与其他 skill 的关系

- 与 `老嗷文风` 同用时，本 skill 优先约束叙述距离、句式、信息进入方式和禁用表达；`老嗷文风` 负责爽点、嘴碎、压场和收益口感。
- 与 `novel-de-ai` 同用时，本 skill 优先处理“装感、冷硬短句、作者腔、出戏比喻”；`novel-de-ai` 负责更广义的去 AI 味和流程打散。
- 与章节细纲冲突时，不改设定收益和关键物件，只调整表达方式、入场顺序和主角贴身感。
