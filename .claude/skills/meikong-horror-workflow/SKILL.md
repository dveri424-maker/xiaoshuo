# 恐怖片无限流创作工作流 Skill

本 skill 是《恐怖片里的违规玩家》的执行卡，不替代正式设定文档。写作或审查前仍需读取：

- `CLAUDE.md`
- `《恐怖片里的违规玩家》设定文档.md`
- `《恐怖片里的违规玩家》剧情大纲与写法定稿.md`

若本 skill 与上述文档冲突，以上述正式文档为准。

## 开工前五问

每次设计副本、章节细纲或正文前，先回答：

1. 强识别物是什么？
2. 当前误导是什么：原作误导、类型误导、系统措辞误导，还是角色常识误导？
3. 代价在哪里出现：死亡、污染、身份替换、记忆缺口、救人付债，还是身体异常？
4. 林扬哪里误判、来不及、救不全或欠债？
5. 违规点落在哪个物件、动作、系统措辞、仪式步骤、站位、称呼或死亡触发条件上？

答不出前三项，不要直接写正文；答不出第四项，容易写成无代价爽文；答不出第五项，容易失去书名辨识度。

## 副本最小结构

副本级硬必备：

- 强识别物。
- 误导。
- 原创隐藏规则。
- 代价验证。
- 可见反制。

副本内必须有，但可延后：

- 林扬误判、救不全或欠债。
- 长期后坐力。

七件事是副本级最低保障，不是每章都要显性打卡。单章只接住其中一到两项。

## 副本卡

设计副本时先填：

```text
副本类型外壳：
强识别物：
主场景：
入场初始信息：
当前误导：
原创隐藏规则：
第一处异常：
第一次死亡/污染/替换/记忆缺口：
林扬第一次误判：
林扬第一次小型反制：
关键配角阻力：
违规点证据：
承载违规点的物件/动作：
可见反噬后果：
通关方式：
长期后坐力：
后续回收点：
```

前两卷的主菜必须有可识别欧美恐怖外壳，原创内容优先作为隐藏规则、误导机制、系统污染或后续回收线索。

## 章节卡

设计单章时先填：

```text
章节标题：
上章钩子：
本章核心问题：
本章主场景：
强识别物/关键物件：
本章恐怖压迫：
本章规则线索：
本章试错代价：
林扬判断：
林扬误判/来不及/救不全/欠债：
违规点证据：
林扬反制动作：
配角恐惧/质疑/分歧：
本章可见成果：
章末钩子：
```

每章至少承担两项功能：建立恐怖压迫、推进规则探索、制造试错代价、强化林扬魅力、推进配角关系、展示违规点动作化反制、增加可见成果、留下章末钩子。

## 审计动作化检查

每次“违规”必须满足：

```text
现场证据
→ 规则矛盾
→ 林扬指出
→ 物件/动作承载
→ 可见后果
```

可见后果可以是门牌变化、照片褪色、档案签名被划掉、仪式中断、鬼被迫退回触发点、任务措辞变化、结算评级异常、违规记录无法清除等。

如果只能写成林扬说“你违规了”，这一场不成立。

## 正文节奏

默认执行：

- 70% 严肃压迫。
- 20% 冷静拆解。
- 10% 黑色幽默。

顺序优先：

1. 先写环境和日常物件的不对劲。
2. 再写配角正常恐惧、误解或自我安慰。
3. 让试探产生伤害、失踪、污染或身份异常。
4. 林扬用一句话切开恐惧，但不能把死亡写轻。
5. 怪物、规则或系统回应他的挑衅。
6. 林扬根据现场信息发现违规点并反制，不靠全知攻略。

## 林扬口吻

林扬可以荒诞、嘴欠、冷静分类，但每句怪话必须服务至少一项：

- 观察异常。
- 缩小规则范围。
- 标记违规点。
- 试探怪物或系统。
- 稳住配角。
- 压制恶人。
- 推动救人、反制或规则反噬。

禁止把他写成李昂低配、游戏主播、纯吐槽役、冷漠实验员、杂学万能怪才或无代价救世主。

## 审查清单

审查副本、章节细纲或正文时逐项检查：

- 是否有可识别欧美恐怖外壳和强识别物。
- 恐怖是否先成立，黑色幽默是否没有冲淡死亡重量。
- 林扬入场是否不知道片名、怪物真实身份和完整规则。
- 原作或类型知识是否只是半真半假的低级情报。
- 是否有原创隐藏规则，而不是照搬原作流程。
- 死亡、污染、替换或记忆缺口是否留下有效信息。
- 林扬是否有误判、救不全、欠债或承担后果。
- 违规点是否来自现场证据，而不是主角硬说。
- 反制是否落在物件、动作、系统措辞、仪式、站位或死亡触发条件上。
- 系统是否通过可见媒介留下痕迹，而不是万能黑箱。
- 章末是否有新异常、新代价、新误判、新系统措辞变化、新债务或新违规记录。
- 本章是否容量过载；如果同时塞进入场、群像、代价、系统解释、规则揭示、新地图入口，应拆章。

## 常用任务提示

副本设计：

```text
使用 meikong-horror-workflow，设计一个《恐怖片里的违规玩家》副本。先回答开工前五问，再填副本卡。重点写强识别物、当前误导、原创隐藏规则、第一次代价、林扬误判、违规点证据、可见反制和长期后坐力。
```

章节细纲：

```text
使用 meikong-horror-workflow，为第XXX章写章节细纲。先回答开工前五问，再填章节卡。不得让林扬全知，不得用系统面板替代现场探索。
```

正文审查：

```text
使用 meikong-horror-workflow 只读审查第XXX章，重点检查恐怖是否成立、死亡是否有重量、林扬是否只嘴欠不行动、规则探索是否像说明书、违规点是否来自现场证据、章末钩子是否有效。
```
