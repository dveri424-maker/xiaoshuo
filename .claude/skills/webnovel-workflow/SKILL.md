# 通用网文小说创作工作流 Skill

本 skill 用于把网文创作任务标准化为：定位、总纲、卷纲、章节细纲、正文、审查、改稿、文档同步。

如果当前项目另有 `CLAUDE.md`、专属 skill、设定文档、卷纲、章节细纲或风格规则，必须同时遵守；若发生冲突，以项目专属规则为准。

## 总原则

1. 先定终点，再设计阶段升级。
2. 主角必须始终是关键判断、关键行动和最终影响的中心。
3. 每一卷、每一组冲突、每一章都要给读者可感知的推进。
4. 章节细纲和正文优先照顾读者代入、场面张力、人物选择、规则逻辑、阶段成果和章末追读。
5. 正文少解释，多用动作、物件、对白、站位、环境变化和后果呈现信息。
6. 大纲服务正文；若正文承载不住，可以拆章、新增章节、合并章节或调整顺序。
7. 调整章节、规则、人物关系、地图、能力、代价或阶段成果后，必须同步相关文档。

## 任务类型

### 新书立项

必须输出：

- 一句话项目定位；
- 主角起点与终点；
- 核心读感和爽点来源；
- 全书终局状态；
- 阶段升级路线；
- 分卷初步规划；
- 每卷卷末大场面、阶段成果、代价或长期影响。

执行重点：不要写成设定堆砌，要写成可持续连载的升级路线。

### 总纲设计

必须明确：

- 主角最终身份或状态；
- 主角最终能力、权限或影响范围；
- 主角最终关系网或可调用资源；
- 核心敌人、核心规则或核心问题的最终失败方式；
- 世界格局或主线秩序如何因主角改变。

### 卷纲设计

每卷必须包含：

```markdown
## 第X卷：卷名

时间/阶段：
核心问题：
主要节点：
- 
- 
- 
卷末大场面：
阶段成果/代价/长期影响：
- 
- 
```

每卷最好完成一次主角跃迁，例如从求生到反制、从观察到制定局部规则、从个人行动到影响一组人或一个场景秩序。

### 章节细纲设计

每章至少包含：

- 标题；
- 上章钩子；
- 本章核心冲突；
- 出场人物；
- 主角关键判断或行动；
- 阻力来源；
- 关键物件或关键信息；
- 关键配角参与点；
- 场面爆点；
- 本章可见成果或代价；
- 章末钩子。

每章至少承担两项功能：推进规则/冲突、制造张力、强化主角魅力、推进关系、展示压迫、增加可见成果或代价、留下后续钩子。

章节细纲必须控制网文章节容量：一章是期待管理单位，不是信息容器。每章最好只围绕一个核心问题和一个主钩子展开，不能把入场、场景展开、人物登场、规则揭示、代价落地、系统解释、下一地图开启等多阶段内容全部塞进同一章。若一章不能用一句话概括，或读者需要同时记住三个以上新问题，应拆章、延后信息或删减功能。

项目专属 skill 可以替换章节卡字段。例如恐怖无限流可以改用副本外壳、强识别物、隐藏规则、试错代价、长期债务等字段。

### 正文续写

写正文时必须：

- 主角始终参与关键判断和关键行动；
- 用动作、物件、对白、站位和后果呈现变化；
- 每章有可见推进和章末钩子；
- 特殊能力、系统或外力不能替代现场行动和人物选择；
- 禁止作者分析腔、策划术语、元叙事和说明书式正文；
- 关系推进必须有可见后果：称呼、信物、站位、行动、隐瞒、选择、代价或后续影响；
- 控制章节容量：正文只解决本章核心问题的一部分，留出下一章期待；强钩子出现后不要立刻被更大的新钩子覆盖；系统提示、规则反转、重大代价和新地图开启要分章接力。

### 正文审查

按以下顺序检查：

1. 字数是否符合项目要求；
2. 主角是否参与关键判断和行动；
3. 是否有读者代入和场面张力；
4. 是否有可见推进、成果或代价；
5. 是否有关键物件或关键信息；
6. 是否有章末钩子；
7. 是否有作者分析腔、策划术语、元叙事；
8. 是否有说明书式正文；
9. 关键配角是否退化为工具人；
10. 特殊能力、系统或外力是否替代了现场行动；
11. 如需拆章或调整细纲，列出需要同步更新的文档。

### 改稿重修

改稿优先级：

1. 补主角关键判断和行动；
2. 补读者代入、场面张力和追读钩子；
3. 把抽象说明改成动作、物件、对白和后果；
4. 删除作者分析腔、策划术语和元叙事；
5. 补关键物件、可见成果或代价；
6. 保持人物关系、伤势、能力、称呼和前后文连续；
7. 如果章节承载不住，不硬压，先改细纲、拆章或新增章节，并同步文档。
8. 如果一章同时承担了三章功能，优先拆分为钩子章、展开章、代价章，而不是继续压缩润色。

## 正文写作规则

### 章节容量与追读接力

正文创作时，章节不是把所有设定一次性交代清楚，而是管理读者期待：接住上章钩子，推进一个明确冲突，留下一个更具体、更危险的新问题。

执行规则：

1. 每章最好能用一句话概括，例如“林扬救下被假父亲敲门的孩子，却被招聘父亲的红门拉走”。如果一句话概括需要列出五六个事件，说明过载。
2. 每章只保留一个主钩子。可以有辅助异常，但不能让系统弹窗、死亡污染、新地图、身份标记、核心规则揭示同时争夺注意力。
3. 上章问题要解决一部分，但不能一次性解决干净；新问题要比上章更具体、更危险。
4. 代价、系统、奖励、身份、地图入口要按章节接力出现。不要在任意一章同时写入场/转场、群像展开、规则验证、系统任务和下一地图。
5. 卷细纲中要标注每章在追读接力中的功能类型，例如钩子章、展开章、探索章、代价章、反制章、揭示章、转场章。相邻章节之间应形成“问题提出—部分解决—新问题升级”的接力，而不是单章自我完成。

### 动作、物件、站位优先

正文不要先解释“这意味着什么”，而要写：

- 谁进门；
- 谁退后；
- 谁按住关键物件；
- 谁想说又停住；
- 谁改变称呼；
- 谁交出钥匙、纸页、录像、门牌、名单或证物；
- 哪扇门被打开或封住；
- 哪个场景被规则改写；
- 哪个人因为主角行动改变选择。

### 禁止作者分析腔

正文禁止出现：

- “本章的作用是”；
- “阶段目标”；
- “剧情功能”；
- “商业爽点”；
- “伏线/伏笔回收”；
- “这意味着”；
- “读者会觉得”；
- “按照设定”。

这些词只能出现在大纲、分析和工作流中，不能进入正文。

### 禁止元叙事

正文人物不能意识到：

- 第几卷；
- 本章；
- 主线；
- 后续剧情；
- 作者安排；
- 剧情节点。

需要表达未来压力时，用世界内可感知物：来信、电话、旧照片、门牌、录音、传闻、敌人动作、钟声、档案、请柬、钥匙、伤口、污染痕迹等。

### 规则、组织或场景秩序变化要动作化

写规则、组织、关系或场景秩序变化时，不能写成说明清单。

至少要有：

- 一个即时冲突；
- 一个阻力来源；
- 一个主角关键动作；
- 一个可见物件或可见后果；
- 一个后续钩子。

把“秩序改变”写成：门被谁锁上，名字被谁划掉，谁把钥匙递出，谁拒绝承认，谁被迫退后，哪个物件从一个人手里转到另一个人手里。

## 人物规则

1. 主角可以受伤、误判、被围攻或被构陷，但不能失格、降智或被配角替代关键判断。
2. 主角每次大决策最好带来具体成果或代价：物件、关系、情报、权限、债务、漏洞、地图入口、名声变化或敌人反击。
3. 主角台词不靠长篇解释，尽量靠动作、局势和一句有功能的话定住场面。
4. 关键配角不能只是工具。应有恐惧、目标、边界、资源、误判、主动选择和后续影响。
5. 关系线必须建立在清醒、主动、可读的选择上；写暧昧、试探、信任、背叛、保护和后果，不写强迫或失去自主权。

## 冲突规则

1. 每个冲突要有可见问题或异常物：尸体、失踪、旧照片、假话、错位声音、烧毁纸页、奇怪痕迹、被撕名单、不该出现的脚印、气味、伤口、污染痕迹等。
2. 每个冲突必须推动主角或主线变化：识别规则、打开地图、改变关系、获得物件、付出代价、制造新敌人或留下长期影响。
3. 特殊能力可以发现方向，但最终最好通过现场动作、人物选择、物件变化、敌人失态或可见后果落地。
4. 反派或规则压力来自局势，不靠强行削主角：信息缺失、时间限制、人质、污染、误导、多人冲突、证物被毁、关系分裂、秘密被逼近等。

## 对话规则

1. 每段关键对话都要有目的：试探、威胁、安抚、遮掩、逼问、甩锅、抢主动权、挖破绽、逼对方让步。
2. 对话要有回弹，不写一问一答式信息传递。
3. 对话中穿插动作和物件：手指停住、杯子偏移、门锁轻响、纸页被捏皱、眼神避开、钥匙串响、椅子后移、灯光闪烁等。
4. 人物口吻要区分：主角稳定但有个人怪味；恐惧者会改口和自我安慰；强势者会遮掩和反问；底层角色更直接；高位角色更克制但有控制欲。

## 路人与环境规则

1. 路人是世界温度计，不是凑人数。
2. 主角行动前后，路人站位、胆量、声音和反应要变化。
3. 可少量复现熟面孔路人，让世界有记忆，但不让路人抢主线。
4. 环境描写必须绑定人物行动、冲突、规则或情绪变化。
5. 环境要落在灯、门、窗、钥匙、纸页、照片、电话、尸体、脚步、血迹、灰尘、气味、墙皮、钟声等可感物上。

## 特殊能力与系统规则

1. 特殊能力可以强，但要有叙事边界：唯一性明确、爽点直观、使用有代价或风险、不能替代主角判断。
2. 如需保密，外人只能感知异常结果，不能公开理解机制。
3. 不写“能力发动”式说明。外显要画面化：纸页变色、空气变冷、声音错位、门牌改字、影子多出一截、伤口变化、物件轻响等。
4. 系统、面板、提示音不能替代现场探索。它们可以误导、遮掩、给副作用或制造新债务。

## 章节自检清单

写完每章后检查：

- 字数是否符合项目要求；
- 主角是否参与关键判断和行动；
- 是否有读者代入和场面张力；
- 是否有可见成果或代价；
- 是否有关键物件或关键信息；
- 物件、关系、权限、债务或规则认知是否发生变化；
- 是否出现作者分析腔、策划术语、元叙事、抽象总结；
- 是否有说明书式正文；
- 章末是否有追读钩子。

## 文档同步规则

正文发生以下变化，必须同步更新相关文档：

- 新增章节；
- 拆章；
- 合并章节；
- 章节顺序调整；
- 关系线提前或延后；
- 新敌人、新规则、新异常或新组织登场；
- 主角提前获得能力、权限、物件或长期债务；
- 某个冲突被改写；
- 某个伏笔被删；
- 某个资源、物件、规则或债务归属变化；
- 卷末成果、代价或长期影响变化；
- 特殊能力、系统规则、地图或时间线变化。

常见同步文档：

- 全书总纲；
- 分卷卷纲；
- 当前卷章节细纲；
- 冲突库；
- 人物关系表；
- 地图与场景设定；
- 风格与禁忌规则；
- 时间线；
- 特殊能力或系统规则；
- 伏笔与回收表。

处理顺序：

1. 保留读者爽感和关键逻辑；
2. 调整章节或细纲；
3. 同步相关设定与时间线；
4. 检查后续章节是否需要顺延；
5. 再继续写正文。
